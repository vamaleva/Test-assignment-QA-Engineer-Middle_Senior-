# Чек-лист проверки файла обмена (информация грузоотправителя)

> Основание: XSD-схема электронного договора перевозки груза (КНД 1110426, версия формата 5.01).
> Чек-лист составлен на основе правил Schematron и структурных ограничений XSD.

---

## 1. Проверка файла и общих реквизитов

| № | Проверка | Источник (правило/элемент) |
|---|---|---|
| 1.1 | Имя файла обмена совпадает со значением `@ИдФайл` | Schematron: `usch:getFileName() = @ИдФайл` |
| 1.2 | `@ВерсФорм` = `"5.01"` | `xs:enumeration value="5.01"` |
| 1.3 | `@КНД` = `"1110426"` | `xs:enumeration value="1110426"` |
| 1.4 | `@НаимДокИнфГО` = `"электронный договор перевозки груза, информация грузоотправителя"` | `xs:enumeration` |
| 1.5 | `@ДатИнфГО` заполнен и соответствует формату `ДД.ММ.ГГГГ` | `ДатаТип` (required) |
| 1.6 | `@ВрИнфГО` заполнен и соответствует формату `ЧЧ:ММ:СС` | `ВремяТип` (required) |

---

## 2. Проверка блока «Сведения о грузе» (`СвГруз` / `ОпГруз`)

| № | Проверка | Источник (правило/элемент) |
|---|---|---|
| 2.1 | `@НаимГруз` обязателен для каждого описания груза | `use="required"` |
| 2.2 | `@ВесГрузБрут` обязателен для каждого описания груза | `use="required"` |
| 2.3 | Если указан `@КолПломб` (в `СвКонтейн`), то обязательно присутствует хотя бы один `НомПломб` | Schematron: `count(@КолПломб)!=0 and count(НомПломб)!=0` |
| 2.4 | Если присутствуют `НомПломб`, то обязательно указан `@КолПломб` | Schematron: `count(@КолПломб)=0 and count(НомПломб)=0` (обратное условие) |

## 3. Проверка блока «Условия перевозки» (`УслПервГруз`)

| № | Проверка | Источник (правило/элемент) |
|---|---|---|
| 3.1 | `@ТипТребСудно` обязателен | `use="required"` |
| 3.2 | `@ПортОтпр` обязателен | `use="required"` |
| 3.3 | `@ПортНазн` обязателен | `use="required"` |
| 3.4 | `@СрокВыпПрвз` обязателен | `use="required"` |
| 3.5 | В блоке `МршПрвз` заполнен хотя бы один из атрибутов: `@МршТекст` или `@МршМашЧит` | Schematron: `count(@МршТекст)!=0 or count(@МршМашЧит)!=0` |
| 3.6 | Если `@ТипЧартер` = `"3"`, то `@ПомщСудно` обязателен | Schematron: `@ТипЧартер=3 → count(@ПомщСудно)!=0` |
| 3.7 | Если `@ТипЧартер` ≠ `"3"`, то `@ПомщСудно` должен отсутствовать | Schematron: `@ТипЧартер≠3 → count(@ПомщСудно)=0` |

## 4. Проверка сведений о подписанте (`ПодпИнфГО`)

| № | Проверка | Источник (правило/элемент) |
|---|---|---|
| 4.1 | Элемент `ФИО` обязателен | `xs:sequence` (minOccurs=1) |
| 4.2 | `@Должн` обязателен | `use="required"` |
| 4.3 | `@СпосПодтПолном` обязателен | `use="required"` |
| 4.4 | Если `@СпосПодтПолном` = `"3"`, то обязательно присутствует `СвДоверЭл` | Schematron: `=3 → count(СвДоверЭл)!=0` |
| 4.5 | Если `@СпосПодтПолном` ≠ `"3"`, то `СвДоверЭл` должен отсутствовать | Schematron: `≠3 → count(СвДоверЭл)=0` |
| 4.6 | Если `@СпосПодтПолном` = `"5"`, то обязательно присутствует `СвДоверБум` | Schematron: `=5 → count(СвДоверБум)!=0` |
| 4.7 | Если `@СпосПодтПолном` ≠ `"5"`, то `СвДоверБум` должен отсутствовать | Schematron: `≠5 → count(СвДоверБум)=0` |

## 5. Проверка сведений об участниках (`СвГО`, `СвПрв`, `СвГП`)

### 5.1. Общие проверки блока `ИдСв`

| № | Проверка | Источник |
|---|---|---|
| 5.1.1 | Выбран ровно один вариант: `СвЮЛУч`, `СвИП`, `СвФЛУч` или `СвИнНеУч` | `xs:choice` |
| 5.1.2 | `Адрес` обязателен | `xs:sequence` (minOccurs=1) |

### 5.2. Юридическое лицо РФ (`СвЮЛУч`)

| № | Проверка | Источник |
|---|---|---|
| 5.2.1 | `@НаимОрг` обязателен | `use="required"` |
| 5.2.2 | `@ИННЮЛ` обязателен (формат: 10 цифр, контрольная сумма) | `use="required"` + `ИННЮЛТип` |
| 5.2.3 | `@КПП` обязателен (формат: 9 знаков, 5-6 знаки: 0-9 или A-Z) | `use="required"` + `КППТип` |

### 5.3. Индивидуальный предприниматель (`СвИП`)

| № | Проверка | Источник |
|---|---|---|
| 5.3.1 | `@ИННФЛ` обязателен (формат: 12 цифр) | `use="required"` |
| 5.3.2 | Элемент `ФИО` обязателен | `xs:sequence` |

### 5.4. Физическое лицо (`СвФЛУч`)

| № | Проверка | Источник |
|---|---|---|
| 5.4.1 | Если `@ИННФЛ` и `@ИныеСвед` оба отсутствуют, то `ФИО` обязательно | Schematron: `count(@ИныеСвед)=0 and count(@ИННФЛ)=0 → count(ФИО)!=0` |
| 5.4.2 | Если `ФИО` отсутствует, то `@ИныеСвед` обязательно | Schematron: `count(ФИО)=0 → count(@ИныеСвед)!=0` |

### 5.5. Иностранная организация/гражданин (`СвИнНеУч`)

| № | Проверка | Источник |
|---|---|---|
| 5.5.1 | Заполнен хотя бы один из атрибутов: `@Идентиф` или `@ИныеСвед` | Schematron: `count(@Идентиф)!=0 or count(@ИныеСвед)!=0` |

## 6. Форматно-логические проверки

| № | Проверка | Тип данных |
|---|---|---|
| 6.1 | `@ИдФайл`: длина от 1 до 255 символов | `xs:string` |
| 6.2 | `@ВерсПрог`: длина от 1 до 40 символов | `xs:string` |
| 6.3 | `ИНН ЮЛ`: ровно 10 цифр, соответствует шаблону | `ИННЮЛТип` |
| 6.4 | `ИНН ФЛ`: ровно 12 цифр, соответствует шаблону | `ИННФЛТип` |
| 6.5 | `КПП`: ровно 9 знаков, соответствует шаблону | `КППТип` |
| 6.6 | `ОГРНИП`: ровно 15 цифр | `ОГРНИПТип` |
| 6.7 | `БИК`: ровно 9 цифр | `БИКТип` |
| 6.8 | Код страны (ОКСМ): ровно 3 цифры | `ОКСМТип` |
| 6.9 | Код субъекта РФ: ровно 2 цифры | `ССРФТип` |
| 6.10 | Почтовый индекс: ровно 6 цифр | `pattern [0-9]{6}` |
| 6.11 | Дата: формат `ДД.ММ.ГГГГ`, календарно корректна (учёт високосных лет) | `ДатаТип` |
| 6.12 | Время: формат `ЧЧ:ММ:СС` (00:00:00 – 23:59:59) | `ВремяТип` |
| 6.13 | `ИдФайлИнфПол` (если указан): длина ровно 36 символов (GUID) | `xs:length value="36"` |
| 6.14 | `@ТипЧартер` (если указан): только `"1"`, `"2"` или `"3"` | `xs:enumeration` |
| 6.15 | `@СпосПодтПолном`: только `"1"`–`"6"` | `xs:enumeration` |
| 6.16 | `@ТипПодпис` (если указан): только `"1"`, `"2"` или `"3"` | `xs:enumeration` |
| 6.17 | `@ИдСтат` (СвИнНеУч): только `"ИО"` или `"ИГ"` | `xs:enumeration` |

## 7. Проверка адреса

| № | Проверка | Источник |
|---|---|---|
| 7.1 | Выбран один из вариантов: `АдрРФ`, `АдрГАР` или `АдрИнф` | `xs:choice` |
| 7.2 | **АдрРФ**: `@КодРегион` и `@НаимРегион` обязательны | `use="required"` |
| 7.3 | **АдрГАР**: `@ИдНом` (GUID, 36 символов) обязателен | `use="required"` |
| 7.4 | **АдрГАР**: `Регион` и `НаимРегион` обязательны | `xs:sequence` (minOccurs=1) |
| 7.5 | **АдрГАР**: Если `Регион` ≠ `99`, то `МуниципРайон` обязателен | Schematron: `Регион!=99 → count(МуниципРайон)!=0` |
| 7.6 | **АдрИнф**: `@КодСтр`, `@НаимСтран`, `@АдрТекст` обязательны | `use="required"` |
| 7.7 | **GLN** (если указан): ровно 13 цифр | `xs:length value="13"` |
