# Документация по процессу анализа и верификации

## 1. Ознакомление с условием задачи
- Получение и анализ исходных требований.
- Определение целей и ожидаемых результатов.
- Выявление ключевых зон внимания и потенциальных рисков.

## 2. Разделение задачи на логические блоки
- Декомпозиция общей задачи на самостоятельные, проверяемые модули.
- Определение взаимосвязей между блоками.
- Приоритизация блоков по критичности и зависимости.

## 3. Распознание диаграммы с помощью ИИ
- Загрузка изображения/схемы в инструмент на базе ИИ.
- Получение текстового описания элементов и связей диаграммы.
- Запись первичной интерпретации данных.

## 4. Ручная верификация данных (что ИИ не наврала)
- Сравнение выходных данных ИИ с исходной диаграммой.
- Проверка точности распознавания сущностей, атрибутов и отношений.
- Фиксация всех расхождений и ошибок ИИ.

## 5. Составление чек-листа по диаграмме
- Создание списка проверок на основе верифицированных данных диаграммы.
- Включение пунктов: полнота элементов, корректность связей, отсутствие лишних объектов.
- Группировка проверок по смысловым блокам.

## 6. Ручная верификация данных (что ИИ не наврала)
- Повторная сверка данных, особенно после составления чек-листа.
- Перепроверка сомнительных или неоднозначных мест диаграммы.
- Окончательное подтверждение достоверности информации.

## 7. Составление отдельных чек-листов по каждому XSD
- Выделение каждого XSD-файла как самостоятельного объекта.
- Для каждого XSD формирование чек-листа, включающего:
  - Пространства имён (namespace)
  - Элементы и их типы
  - Обязательные/необязательные атрибуты
  - Ограничения (minOccurs, maxOccurs, pattern, enumeration и т.д.)

## 8. Ручная верификация данных (что ИИ не наврала)
- Контрольная проверка каждого XSD-описания, сгенерированного ИИ.
- Сопоставление с оригинальными XSD-схемами.
- Исправление выявленных ошибок и неточностей.

## 6. Составление сводного чек-листа по XSD
- Объединение всех отдельных XSD-чек-листов в единый документ.
- Удаление дублирующихся проверок.
- Добавление кросс-проверок между разными XSD (ссылки, импорты, совместимость типов).

## 7. Составление негативных проверок и баг-репорта опираясь на данные из чек-листов по диаграмме и XSD
- Формирование негативных тест-кейсов (нарушение структуры, неверные типы, выход за границы допустимых значений).
- Документирование обнаруженных несоответствий между диаграммой и XSD.
- Оформление баг-репорта с приоритетами и шагами воспроизведения, ссылаясь на пункты чек-листов.
