# Чек-лист проверки файла «Информация о соглашении к электронному договору перевозки груза»

## 1. Проверки корневого элемента (`Файл`)

### Имя и версия файла
- [ ] **Имя файла совпадает с `ИдФайл`**: Проверить, что имя XML-файла (без расширения) равно значению атрибута `ИдФайл`.
- [ ] **Версия формата**: Атрибут `ВерсФорм` равен `"5.01"`.

### Обязательные атрибуты
- [ ] **Заполненность `ИдФайл`**: Атрибут `ИдФайл` присутствует и заполнен (длина от 1 до 255 символов).
- [ ] **Заполненность `ВерсПрог`**: Атрибут `ВерсПрог` присутствует и заполнен (длина от 1 до 40 символов).

---

## 2. Проверки блока «Идентификатор файла грузоотправителя» (`ИдИнфГО`)

> Все атрибуты этого блока **обязательны**.

- [ ] **`ИдФайлИнфГО`**: Заполнен (длина от 1 до 255 символов).
- [ ] **`ДатФайлИнфГО`**: Заполнен, формат `ДД.ММ.ГГГГ`, корректная дата.
- [ ] **`ВрФайлИнфГО`**: Заполнен, формат `ЧЧ:ММ:СС`.
- [ ] **`ЭП`**: Заполнен (длина минимум 1 символ).

---

## 3. Проверки блока «Содержание соглашения» (`СодИнфСглш`)

### Ключевая логика в зависимости от `Функция`

- [ ] **`Функция` = `"1"` (Изменение условий)**:
    - [ ] **Обязателен элемент `СвИзмУсл`**: Он должен присутствовать.
    - [ ] **Отсутствует атрибут `ПричРастрж`**: Его быть не должно.
- [ ] **`Функция` = `"2"` (Расторжение)**:
    - [ ] **Обязателен атрибут `ПричРастрж`**: Присутствует и заполнен (длина от 1 до 1000).
    - [ ] **Отсутствует элемент `СвИзмУсл`**: Его быть не должно.

### Обязательные атрибуты
- [ ] **`УИД_ДМПГ`**: Заполнен (длина от 1 до 36 символов).
- [ ] **`ЛицоСост`**: Заполнен, значение из списка (`"1"` или `"2"`).
- [ ] **`ДатаВстСил`**: Заполнен, формат `ДД.ММ.ГГГГ`, корректная дата.
- [ ] **`НомДПГ`**: Заполнен (длина от 1 до 50 символов).
- [ ] **`ДатаДПГ`**: Заполнен, формат `ДД.ММ.ГГГГ`, корректная дата.

---

## 4. Проверки блока «Изменяемые условия» (`СвИзмУсл`)

> ⚠️ Этот блок проверяется только если `Функция = "1"`.

### Логика для атрибута `ИдПредФайл`
- [ ] **Если `ПорНомСоглш` > 1**: Атрибут `ИдПредФайл` **обязателен** (длина 1-255).
- [ ] **Если `ПорНомСоглш` = 1**: Атрибут `ИдПредФайл` **должен отсутствовать**.

### Обязательные атрибуты
- [ ] **`ПорНомСоглш`**: Заполнен, целое число от `1` до `999`.
- [ ] **`ДатаСоглш`**: Заполнена, формат `ДД.ММ.ГГГГ`, корректная дата.

### Проверки блока «Условия перевозки» (`УслПервГруз`)
- [ ] **Зависимость `ПомщСудно` от `ТипЧартер`**:
    - [ ] **Если `ТипЧартер` = `"3"`**: Атрибут `ПомщСудно` **обязателен** (длина 1-500).
    - [ ] **Если `ТипЧартер` != `"3"`**: Атрибут `ПомщСудно` **должен отсутствовать**.

### Проверка заполненных строковых полей
> Для всех указанных ниже полей, если они присутствуют, проверить, что длина не менее 1 и не более максимальной.

- [ ] `УслПогр` (макс. 1000)
- [ ] `УслВыгрПрич` (макс. 1000)
- [ ] `УслВыгрСклад` (макс. 1000)
- [ ] `УслОчстСудно` (макс. 1000)
- [ ] `ИныеУсл` (макс. 1000)
- [ ] `ПорРасч` (макс. 1000)
- [ ] `РасчНедстчИзлш` (макс. 1000)
- [ ] `ОбязнГО`, `ОбязнПрв`, `УслПрекрОбяз` (макс. 500)
- [ ] `ОтветГО`, `ОтветПрв`, `УслСпор` (макс. 500)

---

## 5. Проверки блока «Подписант» (`ПодпИнфСглш`)

### Логика подтверждения полномочий (`СпосПодтПолном`)

- [ ] **Если `СпосПодтПолном` = `"3"`** (Электронная доверенность):
    - [ ] **Обязателен элемент `СвДоверЭл`**.
    - [ ] **Отсутствует элемент `СвДоверБум`**.
    - [ ] Проверить `СвДоверЭл`:
        - [ ] `НомДовер` (ровно 36 символов).
        - [ ] `ДатаВыдДовер` (формат `ДД.ММ.ГГГГ`).
        - [ ] `ИдСистХран` (длина 1-500).
- [ ] **Если `СпосПодтПолном` = `"5"`** (Бумажная доверенность):
    - [ ] **Обязателен элемент `СвДоверБум`**.
    - [ ] **Отсутствует элемент `СвДоверЭл`**.
    - [ ] Проверить `СвДоверБум`:
        - [ ] `ДатаДовер` (формат `ДД.ММ.ГГГГ`).
        - [ ] `ВнНомДовер` (длина 1-50).
- [ ] **В остальных случаях (`1, 2, 4, 6`)**: Элементы `СвДоверЭл` и `СвДоверБум` **должны отсутствовать**.

### Обязательные атрибуты
- [ ] **`ФИО`**: Элемент присутствует, заполнены `Фамилия` и `Имя` (длина 1-60), `Отчество` опционально.
- [ ] **`Должн`**: Заполнен (длина 1-255).
- [ ] **`СпосПодтПолном`**: Заполнен, значение из списка (`"1"`, `"2"`, `"3"`, `"4"`, `"5"`, `"6"`).

- [ ] **`НаимДокИнфСглш`**: Заполнен, строго равен `"электронный договор перевозки груза, информация о соглашении к электронному договору перевозки груза"`.
- [ ] **`ДатИнфСглш`**: Заполнена, формат `ДД.ММ.ГГГГ`, корректная дата.

## 7. Проверки сложных типов данных (для всех участников)

> Этот раздел применяется везде, где используются соответствующие типы (например, `СвГП`, `Подписант`).

- [ ] **ИНН ЮЛ (`ИННЮЛТип`)**: Ровно 10 цифр, соответствует маске.
- [ ] **ИНН ФЛ/ИП (`ИННФЛТип`)**: Ровно 12 цифр, соответствует маске.
- [ ] **КПП (`КППТип`)**: Ровно 9 символов, соответствует маске (`цифры+цифры+цифры/A-Z`).
- [ ] **ОГРНИП (`ОГРНИПТип`)**: Ровно 15 цифр.
- [ ] **БИК (`БИКТип`)**: Ровно 9 цифр.
- [ ] **Код страны (`ОКСМТип`)**: Ровно 3 цифры.
- [ ] **Код региона (`ССРФТип`)**: Ровно 2 цифры.
- [ ] **Адрес**: Если указан `АдрГАР`, проверить логику:
    - [ ] Если код `Регион` != `"99"`, то элемент `МуниципРайон` **обязателен**.
- [ ] **Иностранная организация (`СвИнНеУч`)**: Обязательно заполнен хотя бы один из атрибутов: `Идентиф` или `ИныеСвед`.
- [ ] **Физическое лицо (`СвФЛУч`)**:
    - [ ] Если отсутствует `ФИО`, то `ИныеСвед` **обязателен**.
    - [ ] Если отсутствуют `ИныеСвед` и `ИННФЛ`, то `ФИО` **обязателен**.
