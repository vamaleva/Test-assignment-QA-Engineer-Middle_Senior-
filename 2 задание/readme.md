# Процесс подготовки чек-листа для тестирования MVP

## 1. Изучила задание
- Проведён анализ исходных требований и технического задания.
- Определены ключевые функции и границы тестирования.
- Выделены основные риски и приоритетные сценарии.

## 2. Попросила ИИ сгенерировать промт
- Сформулирован запрос для искусственного интеллекта.
- В промт включены:
  - описание функциональности;
  - ожидаемые результаты;
  - формат выходных данных (чек-лист).
- ИИ успешно сгенерировал промт на основе предоставленных вводных.

## 3. По сгенерированному промту получила чек-лист проверок
- На выходе получен развёрнутый чек-лист от ИИ.
- Чек-лист включал:
  - позитивные проверки;
  - негативные проверки;
  - граничные значения;
  - проверки интерфейса и интеграций.

## 4. Убрала лишние проверки для тестирования MVP
- Проведён анализ чек-листа на соответствие объёму MVP.
- Удалены:
  - проверки, выходящие за рамки минимально жизнеспособного продукта;
  - сценарии с низким приоритетом;
  - проверки, относящиеся к будущим релизам (Nice to have).
- Сохранены только критически важные для работоспособности и основных пользовательских сценариев проверки.

## 5. Сгенерировала финальный чек-лист
- На основе отфильтрованных проверок составлен итоговый чек-лист для тестирования MVP.
- Финальный чек-лист содержит:
  - чёткие шаги и ожидаемый результат;
  - достаточное покрытие для уверенного запуска MVP.
- Чек-лист готов к использованию в тестовых циклах.
