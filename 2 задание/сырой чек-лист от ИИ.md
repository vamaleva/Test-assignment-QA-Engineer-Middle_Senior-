## 3. Числовые поля (Вес и Объем)

| ID | Тип проверки | Название проверки | Шаги | Ожидаемый результат |
| :--- | :--- | :--- | :--- | :--- |
| F-010 | Граничные | Минимальное граничное значение (0) | Ввести в поле «Вес» значение `0`. | Зависит от логики: поле должно либо пропустить это значение (как вес упаковки?), либо показать ошибку «Значение должно быть больше 0». |
| F-011 | Граничные | Минимальное допустимое значение (1) | Ввести Вес: `1` кг, Объем: `1` м³. | Ошибки нет. Данные принимаются, кнопка активна. |
| F-012 | Граничные | Максимальное граничное значение | Ввести Вес: `20000`, Объем: `82`. | Ошибки нет. Поле принимает значение. Кнопка активна. |
| F-013 | Граничные | Превышение максимума на 1 | Ввести Вес: `20001`, Объем: `83`. | В момент ввода или при потере фокуса появляется сообщение об ошибке (например, «Максимальный вес 20 000 кг»). Кнопка расчета заблокирована. |
| F-014 | Негативная | Ввод отрицательных чисел | 1. В поле «Вес» ввести `-100`.<br>2. Повторить для поля «Объем». | Появляется ошибка валидации. Отрицательные числа не разрешены к отправке. |
| F-015 | Негативная | Ввод нуля с точкой (0.) | 1. Ввести `0.` и уйти с поля (или `0.0`).<br>2. Нажать Tab. | Система должна интерпретировать это как `0` и выдать соответствующую ошибку (если 0 запрещен) либо принять значение (см. F-010). |
| F-016 | Негативная | Ввод текста и спецсимволов | 1. Попытаться ввести `пять`, `123abc`, `@#$%`, `12e` (экспонента). | Физический ввод букв и символов (кроме `.` и `,`) должен быть заблокирован на уровне клавиатуры. Или поле при валидации выдает ошибку «Только числа». |
| F-017 | UX | Разделители дробной части | 1. Ввести вес `10.5` (точка).<br>2. Ввести вес `10,5` (запятая). | Оба значения должны быть приняты корректно (с конвертацией в нужный формат для API). |
| F-018 | Негативная | Отправка пустого поля | Оставить поля «Вес» и «Объем» пустыми. Нажать на соседнее поле. | Подсветка обязательных полей. Текст ошибки «Обязательное поле». |

## 5. Интеграция с API и Кнопка «Рассчитать стоимость»

| ID | Тип проверки | Название проверки | Шаги | Ожидаемый результат |
| :--- | :--- | :--- | :--- | :--- |
| F-023 | Негативная | Отправка пустой формы | Оставить все поля полностью пустыми. Нажать «Рассчитать стоимость». | Запрос к API **не уходит** (проверить через DevTools/Network). Под всеми обязательными полями подсвечиваются ошибки заполнения. |
| F-024 | Негативная | Маршрут не найден (Ошибка 422) | 1. Заполнить форму допустимыми, но не связанными городами (например, откуда нет рейса в точку назначения).<br>2. Нажать кнопку. | Отображается текст ошибки от сервера (например, «По данному направлению перевозки не осуществляются»). Поля формы не сбрасываются для корректировки. |
| F-025 | Негативная | Таймаут соединения / Ошибка 500 | 1. С помощью инструментов разработчика (Network Throttling -> Offline) или заглушки эмулировать обрыв соединения или ошибку 500.<br>2. Нажать кнопку. | Кнопка «залипает» в состоянии загрузки не навсегда. Через N секунд появляется сообщение «Ошибка сервера. Попробуйте позже» или «Сетевая ошибка». |
| F-026 | UX | Блокировка кнопки во время запроса (Debounce) | 1. Заполнить форму валидными данными.<br>2. Включить троттлинг сети (Slow 3G) в DevTools.<br>3. Быстро кликнуть по кнопке 2 раза. | В сетевых запросах виден только **один** POST-запрос. Кнопка после первого клика становится серой/некликабельной и содержит индикатор загрузки (спиннер). |
| F-027 | UI/UX | Возврат кнопки в исходное состояние после ошибки | 1. Смоделировать ошибку (F-025).<br>2. Закрыть сообщение об ошибке. | Кнопка снова активна, текст сменился со спиннера обратно на «Рассчитать стоимость». Данные формы сохранены. |
| F-028 | Позитивная | Формат вывода стоимости | После успешного расчета проверить отображение стоимости. | Текст содержит обозначение валюты (₽). Сумма отформатирована с разделением тысяч (например, 10 500 ₽, а не 10500). |
